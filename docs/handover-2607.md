## 1. 이번 세션에서 추가된 기능

**캘린더 — 입력·조회**
- 전체 일정 검색 (제목·캠페인·메모·작성자·매체)
- 다중 매체 한 줄 등록: `인스타+유튜브+카톡` → 동시 등록 ("1+1 행사" 오인 방지)
- 촬영일정 탭 (유튜브·인스타 전용) + 병기 문법: `7/10 촬영 7/15 업로드 …` 한 줄 →
  촬영 건은 촬영탭, 업로드 건은 매체 캘린더에 각각
- 작성자 자동 기록 (로그인 계정 기반, 수동 입력칸 제거)
- 도시메뉴얼 키워드 확장 (도시·도메), 메모 링크 클릭, 공휴일 표시('26·'27)
- 채널 지칭 제목 자동 제거 2단계: 파서(신규 등록분) + **표시 단계(기존 데이터 포함
  전 화면 — 원본 불변, 수정 폼만 원본 표시)**. 세부 지칭(도시 등)·조사 결합은 보존

**연결 — 탭 간 데이터 흐름**
- 캘린더→스펙 딥링크 ("이 매체 규격·납기 보기", 미스클릭 방지 설계)
- 일정↔SNS 실적 매칭 (인스타 본계정·도시메뉴얼 + 유튜브 4채널) +
  **실적 확정**: 후보 중 담당자가 "선택"하면 그것만 남음
- SNS 지표 추이 (수집마다 스냅샷 누적 → 팔로워·구독 증감 ▲▼)
- 모니터링 하이라이트 (팔로워 급증·급감 / 새 휴면 / 조회 급등 자동 브리핑)

**팀 동기화·감사**
- 변경 이력 (DB 트리거 자동 기록 + 모달 diff + 최근 30일 삭제 기록)
- 알림센터 ("새 변경 N" — 남이 바꾼 일정, 열람=확인)

**공유 — 거버넌스 확정**
- 미러 전용 사이트 (로그인 없는 읽기 전용, VITE_MIRROR=1 빌드 분기) = **사내 타 팀용**
- 외부(대행사·지점) = 미러의 ?view=external — 무계정·새니타이즈. 스펙 탭
  "이 매체 외부 링크 복사" 버튼으로 개별 매체 링크 생성
- **외부인 계정 발급 금지** (뷰어 계정으로도 내부 일정이 읽히는 구멍 — 검토에서 발견·차단)
- 와이드 열람 모드 (스탠바이미용): 폭 1560px·셀당 8건·제목 2줄·기간 중간 연속 표시·
  10분 자동 새로고침. 팀 화면엔 "크게 보기(새 탭)" 버튼. 간트 막대는 보류 확정

**운영 자동화**
- SNS 수집: 격주 전환(비용), 빈 결과 가드, carry-forward(부분 실패 시 계정 유지)
- 캘린더 데이터 주 1회 자동 백업 (git 이력 = 복원점, 0건 저장 방지)

## 2. 사고·복구 기록 (전부 재발 방지 장치 + 회귀 테스트 장착됨)

1. Apify 한도 초과 → 유튜브 데이터 빈 값 덮어쓰기 → 복구 + 빈 결과 가드
2. 크레딧 중도 소진 → 인스타 26계정 소실 → 복구 + carry-forward
3. 격주 cron의 OR 규칙 함정 (실제로는 월 16회 ≈ $27 스케줄) → 주간 cron + 잡 내 게이트로 교체

⚠ **Apify 무료 크레딧 당월 소진** — 리셋 전 수집 재실행 금지 (돌려도 403).

## 8. 신규 기능 제안 백로그 ('26.7.7 — 미착수, 사용자 선택 대기)

기존 제안·기각 항목과 겹치지 않는 완전 신규 후보. 착수 시 add-feature 스킬 절차로.

1. **담당자 디렉토리 탭** — media.js의 담당 정보를 모아 "이 매체 누구한테 연락?"
   원클릭 조회 (팀 내부 + 매체사 창구). 외부 뷰에서는 기존 새니타이즈 규칙 적용. 품 小
2. **매체 계약 관리** — 타겟형 연간 제휴 등 계약 기간·갱신일·조건 기록 + 갱신 임박
   표시. **단가·조건은 미러·외부 완전 숨김 필수** (보안 설계 선행 — 고가 모델). 품 中
3. **일정 코멘트 스레드** — 일정마다 팀원 댓글 (진행 협의가 카톡·구두로 흩어지는 것
   방지, 변경 이력과 별개의 "대화" 기록). DB 테이블 1개 추가. 품 中
4. **매체 성과 카드 (스펙↔실적 연결)** — 스펙 탭 각 매체에 "최근 집행 N건·확정 실적
   요약" 표시. 캘린더↔스펙↔SNS 삼각 연결의 마지막 변. 품 中
5. **중요 일정 핀** — 별표로 상단 고정 (월간 뷰 최상단·와이드 뷰 강조). 품 小
6. **외부 링크 QR 생성** — 개별 스펙 외부 링크를 QR로 (지점 인쇄물·현장 전달용).
   외부 라이브러리 없이 생성 가능. 품 小
7. **이메일 위클리 다이제스트** — 팀즈 불가 대안: 주 1회 이번 주 일정 요약을 사내
   메일로 (GitHub Actions + SMTP). **사내 메일 정책·SMTP 사용 가능 여부 확인 선행**. 품 中
